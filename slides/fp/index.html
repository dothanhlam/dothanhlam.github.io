Facts about FP

Pure functions

Function composition

Shared state

Side effects

Immutability

Pure functions

Produce the same output given the same inputs

Has no side-effects

Function composition

Combining two or more functions to make a new function

Shared state

State which is shared between more than one function

Side Effects

Something changes somewhere you don't know

Pure functions have no side effects

Immutability

An immutable object is an object that can’t be modified after it’s created

FP has no variables

Declarative vs Imperative

Imperative - the flow control describes "How to do"

Declarative - the data flow describes "What to do"

FP - declarative paradigm

the program logic is expressed without explicitly describing the flow control
